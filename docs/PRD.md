作为产品经理，针对这款为合唱团量身定制的“共享多声部简谱云端协同与播放系统”，我为您梳理了 MVP（最小可行性产品）阶段的产品需求文档（PRD）核心内容。在 MVP 阶段，我们的核心目标是 跑通“队长发布-团员获取-分声部独立练习”的最小业务闭环 ，避免陷入繁琐的重度排版功能中。

以下是 MVP 阶段的核心架构梳理：

一、 产品定位与目标
产品愿景 ：成为合唱团数字化排练的基础设施，打破声部练习壁垒。

MVP 核心目标 ：解决传统纸质乐谱版本混乱问题，提供团员可独立控制多声部播放的随身练习工具。

产品形态 ：微信小程序（无需下载安装，利用微信生态实现无缝裂变分享与快速触达）。

二、 MVP 阶段核心功能模块 (3-5个)
1. 用户与合唱团管理模块 (RBAC权限体系)
这是承载协同的基础，重点在于区分队长（管理员）与普通团员的权限边界。

创建与加入 ：队长可通过一键授权登录创建“合唱团”，系统生成专属邀请码或微信分享卡片。团员点击卡片即可直接加入。

声部身份绑定 ：团员加入时，必须选择或由队长分配自己的声部标签（如女高音Soprano、女低音Alto、男高音Tenor、男低音Bass）。

权限隔离 ：队长拥有曲谱的创建、编辑、删除权限；普通团员仅拥有曲谱的只读浏览和本地播放控制权限。

2. 云端简谱发布与单点真实数据源 (Single Source of Truth)
摒弃传统的图片或PDF分享，采用结构化数据流，确保全团始终看到最新乐谱。

可视化基础录入 ：为队长提供基于网页端或小程序的简易录入界面，将简谱数字化为 JSON 格式存储。MVP 阶段支持基础的音高、时值、多声部对齐和歌词输入即可。

实时云端同步更新 ：队长对错音或歌词的任何修改，只要点击保存，云端即刻下发更新指令。

团员端自动拉取 ：团员打开小程序时，系统自动核对版本号并静默拉取最新乐谱，彻底消除“旧版废谱”的干扰。

3. 动态多声部混音与播放控制台 (核心亮点)
这是本产品区别于市面传统排版软件（如作曲大师、番茄简谱）的杀手锏功能，满足“唱自己的声部，听整体和声”的刚需。

基准播放控制 ：提供全局的播放/暂停、节拍器（BPM 速度调节）、光标随音符跳动的视觉跟随功能。

声部独立轨道 (混音台) ：在播放界面下方提供一个类似调音台的 UI 模块。每个声部（包含伴奏轨）拥有独立的音量推子。

独奏 (Solo) 与静音 (Mute) 模式 ：

团员在初学阶段，可以点击自己声部的 Solo ，屏蔽其他声部，专心摸索音准。

在进阶阶段，团员可以点击自己声部的 Mute 或调低音量，让系统播放其他声部作为伴奏，模拟真实合唱环境进行抗干扰训练。

4. 离线缓存与无网练习模块 (Offline-First)
合唱团常常在剧场后台、地下排练室等网络信号极差的环境下使用。

静默缓存 ：只要团员在有网环境下打开过某首曲目，系统后台自动将该曲谱的 JSON 数据及合成所需的音频采样资源缓存至本地 (如 IndexedDB 存储) 。

断网可用性 ：在无网络连接时，团员依然可以完整打开乐谱并使用混音台进行多声部播放练习。

三、 非功能性需求 (NFR)
除了可见的功能，软件的底层质量属性直接决定了用户体验的成败：

类别	需求描述与实现指标
响应速度与延迟 (Performance)	
1. 音频零延迟响应 ：按下播放或拖动声部音量推子时，利用 Web Audio API 机制实现毫秒级响应，杜绝音频卡顿或不同声部间出现相位差错位。

2. 同步低延迟 ：当队长修改乐谱时，网络良好的情况下，团员端需在 1-2 秒内接收到变更提醒。

可用性与稳定性 (Availability)	
1. 跨设备适配 ：基于小程序的响应式设计，必须完美适配各种屏幕尺寸的手机（含全面屏）及平板电脑（如 iPad），保证乐谱在不同宽度下排版不混乱。

2. 后台音频生存权 ：利用微信小程序底层接口，确保用户在锁屏或切回微信聊天界面时，伴奏音乐依然能够稳定保持后台播放。

数据安全性 (Security)	
1. 数据隔离 ：A 合唱团的数据必须与 B 合唱团绝对隔离，非群组成员即使拿到链接也无法查看内部曲谱。

2. 防丢机制 ：队长在编辑曲谱时需具备自动防灾保存功能（结合离线优先策略，即使编辑时突然断网，重新连网后也能将本地修改同步至云端）。

扩展性 (Scalability)	
系统架构在初期（特别是 JSON Schema 数据结构设计时）必须具有前瞻性，以支撑未来可能加入的“智能评价打分”、“一键移调”、“多语言歌词转换”等进阶功能，避免后期因底层数据结构僵化而重构。

产品经理建议： 在 MVP 版本的开发排期中，建议将 60% 的研发精力投入在 “Web Audio API 的多声部混音合成” 与 “小程序数据渲染优化” 这两个技术深水区。至于复杂的制谱排版功能（如高阶连音线、复杂的排版微调），在 MVP 阶段可以做减法，或者考虑在 PC 网页端为队长提供一个相对专业的编辑后台，而小程序端则极致专注于“分发与播放”体验。
