作为产品经理与技术架构设计者，针对“简谱矢量渲染引擎模块”，我们首先需要明确：在前后端分离（或 Serverless）架构下，前端的渲染引擎（如基于 OSMD 的简谱渲染器 ）需要与后端进行标准化的数据交换。RESTful API 的核心职责是 高效传输基于 JSON Schema 的结构化曲谱数据 ，以及 提供服务端静态图形导出能力 。

以下是为您设计的 3 个核心 RESTful API 接口文档。

1. 获取曲谱渲染主数据 (Get Score Schema)
前端在进入排练播放页或编辑页时，调用此接口拉取完整的结构化简谱 JSON 数据，以供前端 Canvas/SVG 引擎实时渲染。

请求路径: /api/v1/scores/{score_id}/schema

请求方法: GET

适用角色: 队长、普通团员

请求参数 (Path Parameters):

score_id (String, 必填): 曲谱的全局唯一标识符 (UUID)。

请求参数 (Query Parameters):

part (String, 选填): 若指定（如 Soprano ），则后端仅返回包含总谱元数据和该单一特定声部的结构，用于减少弱网环境下的传输体积。默认返回总谱。

返回 JSON 示例 (HTTP 200 OK):

{
  "code": 200,
  "message": "success",
  "data": {
    "score_id": "a1b2c3d4-5678-90ef",
    "metadata": {
      "title": "乘着歌声的翅膀",
      "composer": "门德尔松",
      "base_key": "1=G",
      "time_signature": "6/8",
      "default_bpm": 80
    },
    "tracks":
          }
        ]
      },
      {
        "track_id": "t_002",
        "name": "Piano Accompaniment",
        "instrument": "acoustic_grand_piano",
        "measures": [
          // 伴奏轨道数据...
        ]
      }
    ],
    "version": 5,
    "last_updated_at": "2026-04-20T14:30:00Z"
  }
}
2. 保存/更新曲谱渲染数据 (Update Score Schema)
队长在编辑器中对简谱进行任何可视化修改后，前端将最新的 JSON 数据结构提交至此接口，覆盖或合并云端版本。

请求路径: /api/v1/scores/{score_id}/schema

请求方法: PUT

适用角色: 仅限队长 (Admin)

请求 Headers:

Content-Type: application/json

Authorization: Bearer <Token>

请求 Body (JSON): 此处需全量提交修改后的 metadata 与 tracks 数组，结构与 GET 接口的 data 内部结构保持一致。

{
  "metadata": {
    "title": "乘着歌声的翅膀",
    "base_key": "1=G",
    "time_signature": "6/8",
    "default_bpm": 85 
  },
  "tracks": [
    // 更新后的完整音符与歌词数据...
  ],
  "commit_message": "修正了女高音第一小节的附点时值"
}
返回 JSON 示例 (HTTP 200 OK):

{
  "code": 200,
  "message": "Score updated successfully",
  "data": {
    "score_id": "a1b2c3d4-5678-90ef",
    "version": 6,
    "updated_at": "2026-04-27T10:15:30Z"
  }
}
3. 服务端矢量封面生成/导出 (Export Static Render)
虽然应用主要依赖前端引擎进行动态渲染与播放控制，但微信小程序在分享至微信群时，需要一张精美的“曲谱预览封面”。该接口利用 Node.js 端的无头浏览器或服务器端渲染技术（如 OSMD 提供的 Node.js 命令行输出能力 ），将 JSON 数据静默渲染为一张静态矢量图片。

请求路径: /api/v1/scores/{score_id}/export

请求方法: POST

适用角色: 队长、普通团员

请求 Body (JSON):

{
  "format": "png",  // 支持 "svg", "png", "pdf"
  "width": 800,     // 导出图片的宽度分辨率限制
  "target_part": "all", // 渲染目标，"all" 为总谱，或指定 "Soprano"
  "render_theme": "light" // 主题风格，针对微信卡片优化
}
返回 JSON 示例 (HTTP 201 Created):

{
  "code": 201,
  "message": "Export completed",
  "data": {
    "score_id": "a1b2c3d4-5678-90ef",
    "format": "png",
    "download_url": "https://storage.yourdomain.com/exports/a1b2c3d4-preview.png",
    "expires_at": "2026-04-28T10:15:30Z"
  }
}
